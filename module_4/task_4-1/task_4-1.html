Discussion Questions and Answers

1. Why is object-oriented development so important in programming?

Object-oriented development (OOD) makes code modular, reusable, and easier to maintain. It allows developers to model complex systems using objects that combine data and functionality, simplifying collaboration and system design.

2. In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.

The AccountType object centralizes definitions of account types (e.g., Normal, Saving, Credit, Pension). It improves readability, reduces errors, and makes updates easier by replacing hardcoded strings with reusable constants like AccountType.Normal.

This approach ensures consistency and simplifies changes, as updates only need to be made in one place, reducing the risk of typos or mismatches in the code.

3. Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.

Encapsulation hides internal details of a class, exposing only the necessary parts through controlled methods. This improves security, ensures the integrity of data, and makes the class easier to maintain.

To create private members in JavaScript, use the # syntax or closures. Public members can be declared as normal properties or methods within the class.

4. Explain what happens when we use the "new" command in JavaScript.

The new command creates an instance of a class. It allocates memory, sets up the prototype chain, and calls the class's constructor function to initialize the object with any specified properties or methods.
