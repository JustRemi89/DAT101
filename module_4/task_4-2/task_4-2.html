1. What is an array?

An array is a data structure that stores multiple values in a single variable. These values are indexed, starting from 0, and can be of any data type (e.g., numbers, strings, objects). Arrays are useful for organizing data in a sequential manner.

2. What does it mean to "hardcode" in programming?

"Hardcoding" means embedding fixed values directly into the source code instead of using variables, parameters, or external data sources. While simple and quick, it reduces flexibility and maintainability.

3. Explain in plain text what happens in task 3, and how we can traverse (iterate) the array when we (humans) don't "know" how long the array is.

In Task 3, an array is created with pre-defined text split into individual words. Each word is stored as an element of the array. To traverse (iterate) the array without knowing its length, we use its .length property to dynamically control the loop.

4. Why is it beneficial to store instances of a class in arrays, as in task 4?

Storing class instances in arrays helps organize and manage data objects in a structured way. This makes it easier to apply operations like sorting, searching, or modifying the data. For example, in a library system, an array of book instances could be used to store and retrieve information about books.

5. Explain the EWeekDays object from task 7.

EWeekDays is an object that categorizes days of the week with attributes like value and name. Workdays and weekends combine multiple days using binary operations. To add a new category like "WorkoutDays," we can combine specific days using binary operations like OR.

6. In task 8, we will use a "callback" routine when sorting. Explain how such a "callback" routine works.

A "callback" routine is a function passed as an argument to another function and executed at a specific time. In sorting, the sort() method uses a callback to compare two elements, deciding their order based on the result of the comparison.

7. What happens in the sorting routines you have created?

The sorting routine repeatedly compares two elements (a and b) using the callback function. It decides their order by returning a value that indicates whether the first element should come before, after, or stay in the same position relative to the second element.
