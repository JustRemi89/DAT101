  1. In this assignment, we use the DOM. Explain how this library can help us develop interactive web pages.

    2. The DOM bridges HTML and JavaScript, enabling dynamic updates to elements, styles, and structure. It allows developers to respond to user actions like clicks or input, creating interactive and engaging web pages.

  2. What is an event? Simply explain what happens when we add an event listener to an object.

    3. An event is an action detected by the browser, such as a button click or keypress. Adding an event listener ensures that a specific function runs when the event occurs, linking user actions to program behavior.

  3. Sometimes we need to use an event object to get data from the event. Explain how this event object is structured, don't explain all functions or attributes. Try to explain, for example, how to retrieve data from a keyboard or mouse.

    4. The event object contains details like event type, target element, and input data. For example, in a keyboard event, the `key` property gives the key pressed, and in a mouse event, `clientX` provides the cursor's X-coordinate.

  4. Explain the main difference between DOM ID and DOM name. Can we have identical IDs?

    5. An ID uniquely identifies one element and must be unique, while a name groups multiple elements, often in forms. IDs are for single elements; names can apply to multiple for grouping purposes.
